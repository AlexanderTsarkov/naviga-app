# ADR: OOTB v0 — стратегия по диапазону и конфигу радио (radio band strategy)

**Status:** Accepted  
**Date:** 2026-02

---

## Decision

- **OOTB seed/v0 стартует на 433 MHz** (E22/E220-400).
- **Dual-band не делаем.** Допускаем разные SKU по регионам (как у Garmin).
- **Конфиг для пользователя и логики продукта:** радио настраивается через абстракции **Channel + Power (+ SpeedPreset)**; без прямых частот и SF/BW/CR в UI/протоколе.
- **OOTB default power = минимальная мощность**, доступная в выбранном модуле (для Ebyte UART — значение “min power” по командам; фактическая мощность может быть ниже заявленной).
- **Public OOTB profile** и функция **“Back to public network”** (factory reset to defaults) — есть.

---

## Rationale / Consequences

- **Первый рынок — РФ;** PoC на 433; не распыляться на несколько диапазонов в v0.
- **Несовместимость разных band’ов приемлема:** разные SKU для разных регионов; маркировка на корпусе + отображение в приложении (канал/профиль).

---

## Seed vs Prod (note)

- **Seed:** можно держать фиксированный `channel_id=1` для скорости разработки; контракт (Radio v0 spec, HAL) должен поддерживать **channel_id** как поле.
- **Prod OOTB:** поддерживается смена канала через приложение.

---

## Future

- **EU SKU** — вероятно 868/900 MHz.
- **Профессиональные SKU** (150/220 MHz) — возможны как отдельные линейки/версии.
